# UI/UX 구현 상세서 (MVP)
**프로젝트:** AI 기반 공공데이터포털 + 웹정보 수집을 통한 분석 보고서 자동 생성/제공(PDF)  
**문서 버전:** v0.1  
**작성일:** 2026-02-06  
**대상:** PC Web 중심(최소 반응형), 50대 중후반 사용자  
**벤치마크 톤:** 삼성화재 다이렉트(간결한 단계형), 위시켓(깔끔한 정보 구조)

---

## 1. UX 목표 및 원칙(50대 타겟 기준)

### 1.1 목표
- “**한 번의 검색**”으로 핵심 요약을 확인하고, **최소 입력**으로 보고서를 생성해 **PDF로 다운로드**한다.
- 사용자가 불안해하는 지점(정확성/신뢰성/결제/다운로드 제한/수정 제한)을 **UI에서 선제적으로 설명**한다.

### 1.2 핵심 UX 원칙(규칙)
- **한 화면 = 한 개의 Primary CTA**(주 행동 버튼)만 강조한다.
- 전문용어/약어 지양(예: TERM → “검색 제한(악용 방지)”).
- 중요 정보는 “큰 글자 + 카드(요약) + 상세 접기(출처/설명)” 구조를 따른다.
- 실수 방지: 입력/결제/확정 단계에서 **확인(Confirm) 패턴** 필수.
- 장애/실패 대비: “로딩/에러/빈 상태”를 **항상 정의**하고, 재시도/다른 경로를 제공한다.
- 고객센터 없이 운영을 전제로 **도움말(툴팁/FAQ/가이드 문장) UI 내장**.

## 2. 사용자/권한 모델(화면 차등 기준)

### 2.1 사용자 유형
- **비회원(Visitor):** 검색, 개요(프리뷰)까지 접근 가능. “보고서 생성/결제/다운로드/수치조정/자료실”은 제한.
- **회원(Free):** 결제 전 단계(입력, 단계 선택)까지 진행 가능. 결제 완료 시 “보고서 확정/다운로드/자료실(정책에 따라)” 해제.
- **유료결제 사용자(Paid):** 결제 완료한 보고서에 대해 다운로드/확정/1회 조정 가능. 자료실(법령/서식) 접근 가능(정책에 따라).
- **관리자(Admin):** 회원/결제/보고서 로그/자료실 게시물 관리.

### 2.2 정책 고정(의뢰서 반영)
- **다운로드 제한:** 보고서 1건당 **최대 3회(기본값)** 다운로드. (서버 설정값으로 2/3 선택 가능)
- **수치 조정:** 보고서 확정 후 **1회만** 조정 가능.
- **조정 가능 기간:** 결제/생성 시점 기준 **1개월**.
- **검색 항목 변경 제한:** 사용자가 등록한 검색 대상(핵심 항목)은 **1개월 내 변경 불가**(악용 방지).
- **출처 표기:** 요약 카드/보고서 본문에 출처 링크/도메인 표시.

- 드롭다운/팝오버 20

## 6. 컴포넌트 규격(개발·디자인 공통 기준)
> shadcn 컴포넌트를 사용하되, 아래 **사이즈/상태/여백 규칙을 우선**한다.

### 6.1 버튼(Button)
| 구분 | 높이 | 폰트 | 패딩 | 라운드 | 사용처 |
|---|---:|---:|---:|---:|---|
| Primary(L) | 56px | 18px/600 | 0 24px | 12px | “다음/결제/다운로드/확정” |
| Secondary(M) | 48px | 16~18px/600 | 0 20px | 12px | 보조 행동 |
| Tertiary(S) | 40px | 16px/600 | 0 16px | 10px | 테이블/관리자 |

**상태 규칙**
- 기본: Primary 600 배경 + 흰 글자
- Hover: Primary 700
- Disabled: 배경 `#DBEAFE` + 글자 `#64748B`, 커서 금지
- Loading: 스피너 + 버튼 비활성(중복 클릭 방지)

### 6.2 입력(Input / Search / Number)
- 공통 높이: **48px**(기본), **56px(홈 검색 전용)**
- 라벨은 입력 상단(16px), placeholder는 본문(18px)
- 에러 메시지: 입력 하단 14~16px, 빨강(`Error`), 문장형(“필수 항목입니다.”)

**검색 입력(홈) 추가 규칙**
- 좌측 돋보기 아이콘, 우측 “지우기(X)” 버튼
- Enter 키 제출 지원
- 자동완성(선택): 최근 검색 5개 / 추천 검색 5개

**숫자/금액 입력 규칙**
- 우측 정렬 + 천 단위 콤마 표시
- 단위(원/%)는 입력 우측 suffix로 분리 표시(값과 분리)

### 6.3 카드(Card)
- 기본: 흰색 Surface, 보더 1px, 라운드 16px, 패딩 24px
- “요약 KPI 카드” 구성:
  - 라벨(16px, muted)
  - 값(28px, 700, tabular-nums)
  - 보조설명(16px)
  - “출처 보기” 링크(16px)

### 6.4 모달(Dialog)
- 기본 폭: 560px, 모바일은 풀스크린
- 구조: Header(제목+닫기) / Body / Footer(버튼 1~2개)
- 닫기 규칙: ESC, 외부 클릭(결제 등 중요한 단계는 외부 클릭 금지)
- 포커스 트랩 필수

### 6.5 스텝퍼(Progress Stepper)
- 단계: 최대 5단계로 단순화
- 현재 단계 강조(Primary), 완료는 체크 아이콘
- 예시 단계명:
  1) 검색
  2) 입력
  3) 단계 선택
  4) 결제
  5) 확정/다운로드

### 6.6 알림(Toast/Alert)
- 상단 고정 배너(Alert): 중요한 정책(다운로드 제한/1회 조정) 안내
- 토스트(Toast): 저장 완료/오류/재시도 안내(3~5초, 수동 닫기 가능)

### 6.7 테이블(관리자)
- 헤더 44px, 행 48px
- 열 정렬: 숫자 우측, 텍스트 좌측
- 필터/검색은 테이블 상단에 고정(1줄)
- 페이지네이션: 우측 하단, 큰 버튼(48px 높이)

## 7. 핵심 사용자 플로우(화면별 명세)

### 7.1 홈/검색
**목표:** 사용자가 “무엇을 입력해야 하는지” 즉시 이해  
**레이아웃:** 중앙 정렬(폼 컨테이너 720px), 상단 로고/간단 메뉴

**필수 요소**
- 큰 검색 입력(56px) + Primary 버튼 “분석 시작”
- 예시 안내(클릭 가능): “예: 상호명 / 소재지번”
- 검색 제한 안내(TERM):
  - “오늘 남은 검색: N회”
  - “다음 검색 가능: 00:30 후” (쿨다운 적용 시)
- 신뢰 문구(짧게): “공공데이터와 웹 공개정보를 기반으로 작성됩니다.”

**상태**
- 로딩: 버튼 disabled + 진행 페이지로 이동
- 입력 오류: “검색어를 입력해 주세요.”

### 7.2 검색 진행(로딩)
**목표:** 기다림을 불안하지 않게
- 4단계 진행바:
  1) 공공데이터 조회
  2) 웹 정보 수집
  3) 비교군 분석/계산
  4) 보고서 문장 생성(출처 기반)
- 평균 소요 안내(예: “약 30~60초”)
- 실패 시: “일시적으로 정보를 가져오지 못했습니다. 잠시 후 다시 시도해 주세요.” + 재시도 버튼

### 7.3 개요(프리뷰)
**목표:** 결과를 ‘카드 3개’로 즉시 이해

**필수 KPI 카드(의뢰서 반영)**
- 현재 백분율
- 단가(원/단위)
- 솔루션 사용 진행 여부(예/아니오 + 설명)

**출처 표시**
- 각 KPI 카드에 “출처 보기” → 모달에 도메인/URL(가능하면) + 수집 시각 표시

**CTA**
- Primary: “보고서 만들기(결제 전)”
- Secondary: “다른 항목 검색”

**유료/비유료 차등(표시 규칙)**
- 비회원: “보고서 만들기” 클릭 시 로그인/가입 유도 모달
- 유료 사용자: “내 보고서에서 이어서”

### 7.4 결과 없음(DB 미존재)
- 안내: “해당 항목은 데이터베이스에서 찾지 못했습니다.”
- 대체 경로(2개 중 1개 Primary만)
  - Primary: “수기 입력으로 진행”
  - Secondary: “다른 검색어로 다시 검색”
- 이유/팁(접기): 표기 형식 예시 제공

### 7.5 로그인(소셜)
- 버튼(56px): 카카오 / 네이버 (필수), 기타는 옵션
- 설명(짧게):
  - “다운로드 횟수 제한(최대 3회)과 1회 조정 기능을 위해 로그인해 주세요.”
- 실패 시: “로그인에 실패했습니다. 다시 시도해 주세요.”

### 7.6 수기 입력(1페이지 폼)
**목표:** “딱 한 장”에서 끝나는 느낌

**구성 규칙**
- 섹션을 3개로 나누되 접기 금지(스크롤은 허용)
  1) 필수 정보
  2) 선택 정보
  3) 확인/동의(있다면)
- 필드 수는 MVP에서 12~18개 이내 권장
- 각 필드 아래 예시/단위/오류 문구 제공

> 필드 리스트는 엑셀 로직 확정 시 최종 고정(본 문서에서는 placeholder로 유지)

### 7.7 단계(1/2/3) 선택
**목표:** 비교를 어렵지 않게(3개 카드 + 표 1개)
- 카드 3개(동일 규격):
  - 제목: 1단계/2단계/3단계
  - 요약(2줄 이내)
  - 예상 합계 금액(큰 숫자)
  - “추천” 배지(1개만)
- 하단 비교표(선택): 포함 항목 체크리스트

CTA: “이 단계로 진행하기”

### 7.8 결제(건당 결제)
- 결제 요약 카드: 검색 항목/선택 단계/가격
- 약관 동의(체크박스 2~3개, 링크 포함)
- 세금계산서 정보(필요 시): 사업자번호 등 최소 입력
- Primary: “결제하기”
- 결제 실패: 이유 + “다시 시도” + “다른 결제수단” (가능 시)

### 7.9 보고서 생성(진행)
- 로딩 단계 표시 + “생성 중에는 창을 닫아도 내 보고서에서 확인할 수 있습니다.”
- 완료 시 자동 이동: 보고서 프리뷰/확정

### 7.10 보고서 프리뷰/확정(1회 조정)
**목표:** “확정” 의미를 명확히
- 상단 경고 배너(항상 표시):
  - “확정 후 **1회만** 수치 조정이 가능합니다.”
  - “조정 가능 기간: 생성일로부터 1개월”
- 프리뷰는 섹션 단위(접기 가능)로 제공:
  - 요약
  - 지표/표
  - 비교군 분석
  - 출처 목록

**조정 가능 영역 UI 규칙**
- 변경 가능한 값만 “연필 아이콘” 표시
- 편집 모드 진입 시: 입력 필드로 전환 + 단위 표시 + 취소/저장
- 저장 시: “이번 저장이 1회 조정으로 처리됩니다. 진행할까요?” 확인 모달
- 확정 CTA: “확정하고 PDF 다운로드”

### 7.11 다운로드(횟수 제한)
- “남은 다운로드: 3/3”을 카드 상단에 크게 표시
- 다운로드 버튼(Primary) 56px
- 다운로드 이력(간단): 날짜/시간/남은 횟수
- 제한 소진 시:
  - 버튼 disabled + 안내 “다운로드 횟수를 모두 사용했습니다.”
  - 대체: “내 보고서에서 화면으로 보기”(옵션)

### 7.12 내 보고서 목록
- 리스트 카드/테이블(사용자용은 카드 권장):
  - 제목(검색 항목)
  - 생성일
  - 상태(생성중/완료/오류)
  - 남은 다운로드
  - 조정 여부(미사용/사용완료)
- Primary: “보고서 열기”

### 7.13 자료실(법령/서식, 유료 전용)
- 상단 카테고리 탭
- 검색 입력(48px)
- 게시물 카드: 제목/요약/업데이트일
- 접근 제한 시 Paywall:
  - “유료 회원만 이용 가능합니다.” + “결제/멤버십 안내” CTA

## 8. 관리자 UI 명세(밀도↑, 가독성 유지)

### 8.1 레이아웃
- 좌측 사이드바 280px(아이콘+텍스트), 상단 바 56px
- 본문 테이블 중심, 필터는 상단 1줄 고정

### 8.2 필수 화면
- 대시보드: KPI 카드(오늘 결제/보고서 생성/오류율/활성 사용자)
- 회원 관리: 검색/필터(등급), 상세 보기 모달(기본 정보/결제 이력)
- 결제 내역: 상태(성공/실패/환불), 영수증/세금계산서 관련 링크
- 보고서 로그: 상태/소요시간/에러 메시지(복사 버튼)/재시도 트리거(권한자)
- 자료실 관리: 게시물 CRUD, PDF 업로드, 노출/비노출 토글

## 9. 상태/에러/빈 화면 규칙(필수 구현)
- **로딩:** 스켈레톤(카드/표) + 진행 문구 + 취소/뒤로(가능 시)
- **빈 상태:** “데이터가 없습니다” + 다음 행동(추가/검색/필터 초기화)
- **오류:** 사용자 행동 기준의 문장 + 재시도 + 대체 경로
- **네트워크/외부 API 실패:** “일시적 오류”로 통일, 기술 메시지 노출 금지(관리자 로그에만)
- **TERM 제한:** 남은 횟수/쿨다운 타이머를 화면 상단에 고정 배지로 표시

## 10. 접근성(A11y) 체크리스트(개발 수용 기준)
- 키보드 탭 이동 가능(모달 포커스 트랩 포함)
- 포커스 링 항상 표시(삭제 금지)
- 폼 요소는 `label` 연결 및 오류 메시지 연계
- 색만으로 상태 구분 금지(아이콘/텍스트 병행)
- 최소 폰트 16px, 본문 18px 준수
- 모바일(375px)에서 가로 스크롤 발생 금지

## 11. QA/검수 기준(완료 정의)
- (흐름) 검색 → 개요 → 입력 → 단계 선택 → 결제 → 생성 → 확정/1회 조정 → 다운로드(3회)까지 단절 없음
- (정책) 조정 1회 제한/1개월 제한/다운로드 제한이 UI에서 항상 명확
- (신뢰) 모든 핵심 지표/요약에 출처 확인 경로 존재
- (가독성) 50대 기준: 버튼/입력/텍스트 크기 충분, 정보 과밀 없음
- (반응형) 375/768/1024/1440 기준 레이아웃 깨짐 없음

## 12. 용어/문구 스타일 가이드(간단)
- 존댓말 통일(“하세요/됩니다”)
- 짧게 쓰기: 한 문장 25자 내외 권장
- 금액/단위: “원”, “%” 명확히 표기
- 예시 문구
  - 저장 완료: “저장했습니다.”
  - 오류: “잠시 후 다시 시도해 주세요.”
  - 제한 안내: “남은 다운로드: 2회”

## 13. 추후 확장(명세 범위 밖, 구조만 고려)

## 3) 구현/검수 실행 계획(문서 기반으로 실제 UI 만들 때)
1) `상세서.md` 기준으로 **디자인 토큰(Figma Styles + Tailwind theme)** 동기화
2) 공용 컴포넌트(버튼/입력/카드/모달/스텝퍼/토스트/테이블)부터 구축
3) 사용자 플로우(홈→개요→입력→선택→결제→확정→다운로드) 화면을 순서대로 연결
4) 정책 제한(TERM/1회 조정/다운로드 제한/1개월) UI 상태를 서버 파라미터로 주입 가능하게 구현
5) 접근성/반응형 QA 체크리스트로 검수(375/768/1024/1440)
